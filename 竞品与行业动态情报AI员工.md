# AI员工需求：竞品与行业动态情报AI员工

## 0. 为什么需要这个AI？
- **现状痛点**：需要手动关注美的、沁园、安吉尔等竞品的动态，浏览多个公众号、新闻网站、企业官网，每次耗时2-3小时，且容易遗漏重要信息
- **参考案例**：类似Bloomberg的行业监控系统，自动追踪竞争对手和行业新闻
- **预期价值**：节省80%情报收集时间，及时发现竞品新品发布、价格调整、技术突破、行业政策变化

---

## 1. 一句话定义
> 当净水器行业出现新动态时，这个AI每两周自动爬取竞品公众号、行业新闻、竞品官网，使用AI分析整理后生成结构化情报报告，帮助管理者了解市场动向，省去手动搜集和阅读的时间。

## 5. 边界与限制（防呆设计）

### 异常情况处理
- **如果某个公众号改名或停更** → 标记"该公众号14天无更新"，提示人工检查是否需要调整监控列表
- **如果爬取失败（网络问题/反爬虫）** → 使用上次缓存数据 + 标注"数据不完整"，3次失败后发送告警
- **如果AI误判情报重要性** → 人工反馈后，AI学习调整（如"竞品招聘信息"不应标为高优先级）
- **如果同一条新闻被多个渠道重复抓取** → 基于标题相似度去重，保留最完整的版本

### 安全限制
- 处理超时限制：单次情报收集 < 30分钟（超时则终止）
- 爬取频率限制：每个网站请求间隔 5-10秒，使用代理IP池防封禁
- 敏感信息过滤：报告中不包含竞品的内部机密（如泄漏的财务数据），只分析公开信息
- 数据存储：原始数据保留3个月，分析报告永久保存

### 防止误报机制
- **关键词白名单**：明确哪些关键词必须关注（如"降价""融资""新品"）
- **竞品黑名单**：过滤掉非直接竞品（如家电综合品牌的冰箱新闻）
- **时效性检查**：超过14天的旧闻不纳入本期报告（除非是重大政策）

## 6. 技术约束

### 性能要求
- **预期响应时间**：单次情报收集完成 < 30分钟（包含爬取、分析、生成报告）
- **并发支持**：3个渠道（公众号、官网、新闻）并行爬取
- **数据存储**：保留最近12期（6个月）的情报数据，便于趋势分析

### 依赖资源
- **外部服务**：
  - 爬虫工具：Playwright/Selenium（处理动态页面）
  - 代理IP池：防止被封禁（如快代理、阿布云）
  - LLM API：Claude/GPT-4（情报分类、分析、摘要）
  - OCR服务：如竞品发布图片版PPT，需OCR提取文字
- **本地资源**：
  - 竞品信息库：`D:\intelligence\competitors\`（存储竞品基础信息）
  - 监控配置文件：`D:\intelligence\config.json`（公众号、关键词、网站列表）
  - 报告归档：`D:\intelligence\reports\{date}\`
- **API权限**：无需平台官方API（使用爬虫）
- **成本预估**：
  - 代理IP：¥50/月
  - LLM分析：每期约100条内容 × ¥0.05 = ¥5元/期
  - 月运行成本约 ¥60元（每月2次）

### 技术栈建议
- 后端：Python（便于集成爬虫和AI）
- 爬虫框架：Playwright（支持微信公众号等复杂页面）
- 数据处理：pandas（数据清洗、去重、分组）
- AI调用：LangChain（批量处理和结构化输出）
- 报告生成：Jinja2（HTML模板）+ pdfkit（生成PDF）
- 数据存储：PostgreSQL（全文搜索）+ Elasticsearch（关键词索引）
- 定时任务：APScheduler

## 7. 如何判断做对了？（验收标准）

### 定量指标
- 情报召回率 > 90%（真实重要情报必须被检出）
- 误报率 < 15%（被标为"重要"的情报中，真正重要的占比 > 85%）
- 响应时间：从爬取到发送报告 < 30分钟
- 系统可用性 > 95%（每两周任务成功执行率）

### 定性检查
- 用真实竞品新品发布测试，必须被标记为"高优先级"且分析合理
- 用通用新闻（如"某地自来水管道改造"）测试，不应误判为重要情报
- 用行业政策测试，必须提取出正确的实施日期和影响分析
- 报告让管理者能"看完就知道市场发生了什么"，无需再去翻原文

### 最小可行版本（MVP）
第一版只需实现：
- [x] 爬取3个竞品公众号 + 2个竞品官网 + 3个行业关键词的新闻
- [x] AI分类筛选出重要情报（竞品动态、行业政策）
- [x] 生成结构化报告（JSON + HTML邮件）
- [x] 发送邮件通知
- [ ] 暂不实现：附件下载（如竞品PPT）
- [ ] 暂不实现：趋势分析（需积累3期以上数据）

## 8. 后续迭代方向

### V2：深度分析（1个月后）
- 竞品价格监控：自动抓取京东/天猫的竞品价格变化，分析价格策略
- 竞品社交媒体监控：扩展到微博、小红书、抖音等平台
- 自动生成竞品对比表：将朴道与主要竞品的产品参数、价格、卖点制成对比表

### V3：趋势分析（3个月后）
- 积累6期（3个月）数据后，AI自动生成趋势报告
  - 哪个竞品最活跃（发布频率、新品数量）
  - 行业热点趋势（哪些关键词出现频率上升）
  - 竞品策略分析（价格走势、渠道变化）
- 预测性分析："根据美的近期动作，预测其下季度可能推出家用纳滤产品"

### V4：智能应对建议（6个月后）
- 不仅分析情报，还自动生成应对方案
  - 竞品降价 → AI建议朴道的定价调整方案（含利润测算）
  - 新政策发布 → AI自动生成合规自查清单
  - 竞品新品发布 → AI生成对标产品的PRD初稿
- 与公司OA系统打通，自动创建待办任务

## 4. 处理流程（伪代码/流程图）

```
1. 定时触发（每两周一次，周一 09:00）
   └─ 启动情报收集任务

2. 数据采集阶段（并行执行）
   ├─ 【渠道A】爬取竞品公众号
   │   ├─ 使用微信搜狗搜索打开目标公众号（如"美的净水"）
   │   ├─ 获取最近14天发布的文章列表
   │   ├─ 提取：标题、发布时间、阅读数、正文、配图
   │   └─ 保存到 raw_data_wechat[]
   │
   ├─ 【渠道B】爬取竞品官网动态
   │   ├─ 访问竞品官网的新闻中心（如 https://water.midea.com/news/）
   │   ├─ 提取最近14天的公告/新闻
   │   └─ 保存到 raw_data_website[]
   │
   └─ 【渠道C】爬取新闻网站（关键词搜索）
       ├─ 在今日头条/腾讯新闻搜索行业关键词
       │   └─ 关键词："净水器新国标"、"直饮水政策"、"DPM技术"等
       ├─ 筛选发布时间：最近14天
       ├─ 提取前50条新闻
       └─ 保存到 raw_data_news[]

3. 数据清洗与去重
   ├─ 合并三个渠道的原始数据
   ├─ 去重（基于标题相似度 > 80%）
   ├─ 过滤无关内容（如纯广告、招聘信息）
   └─ 生成 clean_data[]

4. AI初筛：识别重要情报
   ├─ 遍历 clean_data[]
   ├─ 调用LLM判断相关性和重要性
   │   └─ Prompt: "你是净水器行业分析师。判断以下内容是否与净水器行业相关，
   │       以及对朴道水汇（主营DPM纳滤技术商用净水器）的重要性。输出JSON：
   │       {
   │         'is_relevant': true/false,
   │         'category': '竞品动态/行业政策/行业动态/技术突破/其他',
   │         'priority': '高/中/低',
   │         'reason': '判断依据',
   │         'key_entities': ['涉及的公司/产品/人物']
   │       }"
   ├─ 筛选 is_relevant = true 且 priority ≠ '低' 的内容
   └─ 保存到 important_data[]

5. AI深度分析（针对重要情报）
   ├─ 遍历 important_data[]
   ├─ 对每条情报调用LLM深度分析
```
